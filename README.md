## 项目介绍

Academic Hub V6 以「**贞德·达尔克**」为设计灵魂，打造兼具学术严谨性与视觉美感的科研工作台。平台采用 **React 19 + TypeScript** 前端、**Edge Functions + Cloud Functions** 边缘计算后端、**KV Storage** 边缘数据库的现代化架构，部署于腾讯云 EdgeOne Pages 全球边缘网络。

**设计理念**：以灰蓝（`#3d5a80`）+ 金棕（`#c9a04d`）为主色调，浅灰白（`#f0f2f5`）为背景，营造沉稳专注的学术氛围。

## 解决的问题

| 问题 | Academic Hub 的解决方案 |
|------|---------------------------|
| 论文散落各处，缺乏统一管理系统 | 集中管理所有论文，支持 arXiv 搜索、DOI 导入、文件上传 |
| 文献阅读进度难以追踪 | 阅读状态管理（未读/在读/已完成）+ 阅读热力图可视化 |
| 缺乏智能 AI 辅助分析 | 集成 LLM API，支持论文解析、学术问答、流式对话 |
| 项目管理与论文脱节 | 项目关联论文，进度追踪，目标管理 |
| 文献库组织混乱 | 多文献库管理，支持颜色/图标自定义，批量操作 |
| 缺乏知识图谱可视化 | Canvas 知识图谱，力导向布局 + 子图卡片 |
| 部署复杂，运维成本高 | EdgeOne Pages 一键部署，Git 推送自动构建 |

## 功能特性

### 📊 仪表盘（Dashboard）

- **统计概览**：论文总数、收藏数、本月阅读、进行中项目
- **阅读热力图**：GitHub 风格 365 天热力图，连续活跃火焰动画
- **最近论文**：快速访问最近添加/更新的论文
- **进行中项目**：项目进度追踪，完成目标可视化

### 📚 论文管理（Library）

- **论文列表**：支持搜索、排序、筛选（收藏/已读/未读）
- **论文详情**：标题、作者、摘要、关键词、引用信息
- **笔记系统**：富文本笔记，支持多笔记管理
- **高亮标注**：PDF/文本高亮，颜色标记
- **收藏管理**：一键收藏/取消收藏

### 🗂️ 我的文献库（My Library）

- **多文献库管理**：创建多个文献库，自定义颜色和图标
- **论文组织**：将论文添加到不同文献库
- **批量管理**：批量选择、批量删除、批量移动、批量导出（BibTeX/CSV）、批量标签
- **搜索筛选**：按标题、作者、期刊搜索论文

### 🔬 研究项目（Research）

- **项目管理**：创建、编辑、删除研究项目
- **进度追踪**：项目状态（规划中/进行中/已完成）、目标完成度
- **论文关联**：项目关联相关论文
- **项目详情**：描述、目标、时间线

### 🤖 AI 智能对话（AI Chat）

- **流式对话**：SSE 流式响应，实时显示 AI 回复
- **论文解析**：上传论文 PDF/文本，AI 自动解析元数据
- **学术问答**：基于论文内容的智能问答
- **对话管理**：多轮对话历史，上下文记忆

### 📥 导入导出（Import & Export）

- **arXiv 搜索**：搜索 arXiv 论文，一键导入
- **Semantic Scholar 搜索**：搜索 Semantic Scholar 论文
- **批量导入**：从搜索结果批量导入论文
- **导出功能**：导出为 BibTeX、CSV、JSON 格式

### 📁 资料管理（Materials）

- **资料库**：管理研究资料（文档、笔记、链接）
- **分类管理**：按类型（文档/笔记/链接/代码）分类
- **收藏功能**：收藏重要资料
- **全文搜索**：搜索资料标题和内容

### 🕸️ 知识图谱（Knowledge Graph）

- **Canvas 可视化**：交互式知识图谱
- **力导向布局**：节点自动布局，拖拽交互
- **子图卡片**：点击节点查看论文/项目详情
- **关系展示**：论文-项目-作者关联关系

### ⚙️ 设置（Settings）

- **个人资料**：用户名、显示名、邮箱、机构
- **界面设置**：主题（浅色/深色/系统）、语言
- **引用格式**：APA/MLA/Chicago/GB7714
- **通知设置**：邮件通知、浏览器通知

### 👩🏫 管理后台（Admin）

- **用户管理**：查看所有用户，禁用/启用账号
- **数据统计**：系统总用户数、论文数、项目数
- **内容管理**：管理所有论文和项目
- **系统监控**：查看系统运行状态

## 优劣势分析

### ✅ 优势

| 优势 | 说明 |
|------|------|
| **边缘计算架构** | Edge Functions 全球部署，亚太区域延迟 < 50ms |
| **现代化技术栈** | React 19 + TypeScript + Vite 8，开发体验优秀 |
| **全栈 TypeScript** | 前端到后端类型安全，减少运行时错误 |
| **AI 集成** | 内置 LLM API 集成，支持流式对话和论文解析 |
| **知识图谱** | 独特的 Canvas 知识图谱可视化，直观展示研究关联 |
| **批量操作** | 强大的批量管理功能，提升文献整理效率 |
| **阅读追踪** | 阅读热力图 + 统计，量化研究投入 |
| **一键部署** | EdgeOne Pages Git 推送自动部署，无需运维 |
| **Mock/Real 双模式** | 开发环境使用 Mock 数据，生产环境切换真实 API |
| **响应式设计** | Tailwind CSS + shadcn/ui，完美适配桌面和移动端 |

### ⚠️ 劣势与限制

| 限制 | 说明 | 缓解方案 |
|------|------|----------|
| **KV 存储限制** | KV Storage 不支持复杂查询，适合键值读写 | 数据模型设计为扁平化键值结构 |
| **Edge Functions 超时** | V8 Runtime 最大 500ms CPU 时间 | AI 等长时间任务使用 Cloud Functions |
| **无关系型数据库** | 不支持 SQL 联表查询 | 数据冗余设计，应用层 JOIN |
| **单租户架构** | 当前为单用户实例，多用户数据隔离靠 USER_ID 前缀 | 可扩展为多租户 SaaS |
| **文件存储受限** | EdgeOne KV 不适合大文件存储 | 大文件建议使用对象存储（COS） |
| **AI 依赖外部 API** | AI 功能需要有效的 LLM API Key | 提供 Mock AI 模式用于开发测试 |

## 完整使用说明书

### 一、首次使用

#### 1. 注册账号

1. 访问平台首页，点击「注册」
2. 输入用户名（3-20 字符）、邮箱、密码（至少 6 位）
3. 系统实时检查用户名可用性
4. 注册成功自动登录

#### 2. 登录

- **管理员**：`admin` / `123456`
- **普通用户**：使用注册的用户名/密码登录

### 二、论文管理

#### 1. 添加论文

**方式一：arXiv 搜索**
1. 进入「导入导出」页面
2. 选择「arXiv 搜索」标签
3. 输入关键词，点击搜索
4. 浏览搜索结果，点击「导入」将论文添加到库

**方式二：Semantic Scholar 搜索**
1. 进入「导入导出」页面
2. 选择「Semantic Scholar」标签
3. 输入关键词搜索
4. 选择论文，批量导入

**方式三：手动创建**
1. 进入「文献库」页面
2. 点击「添加论文」
3. 填写论文信息（标题、作者、期刊、年份等）
4. 点击「保存」

#### 2. 查看论文详情

1. 在论文列表中点击论文标题
2. 查看完整信息：标题、作者、摘要、关键词、引用
3. 添加笔记、高亮标注
4. 标记阅读状态（未读/在读/已完成）

#### 3. 管理笔记

1. 在论文详情页，点击「添加笔记」
2. 输入笔记内容（支持 Markdown）
3. 保存笔记
4. 可添加多条笔记，每条独立管理

#### 4. 高亮标注

1. 在论文详情页，选中文本
2. 点击「高亮」按钮
3. 选择高亮颜色
4. 高亮内容自动保存

---

### 三、文献库管理

#### 1. 创建文献库

1. 进入「我的文献库」页面
2. 点击「创建新文献库」
3. 输入名称、描述
4. 选择颜色和图标
5. 点击「创建」

#### 2. 添加论文到文献库

**方式一：从文献库添加**
1. 进入目标文献库
2. 点击「添加论文」
3. 从论文列表选择要添加的论文
4. 确认添加

**方式二：从论文详情页添加**
1. 打开论文详情页
2. 点击「添加到文献库」
3. 选择目标文献库
4. 确认

#### 3. 批量管理

1. 进入「我的文献库」页面
2. 点击「批量管理」按钮
3. 勾选需要操作的论文
4. 使用底部操作栏：
   - **删除**：批量删除选中论文
   - **移动**：移动到其他文献库
   - **BibTeX**：导出为 BibTeX 格式
   - **CSV**：导出为 CSV 格式
   - **添加标签**：批量添加标签
   - **移除标签**：批量移除标签
5. 点击「退出批量」结束批量模式

#### 4. 编辑/删除文献库

1. 在文献库列表中，点击文献库右侧的「编辑」按钮
2. 修改名称、描述、颜色、图标
3. 点击「保存」
4. 点击「删除」可删除文献库（默认文献库不可删除）

## 部署指南

### 方式一：使用 EdgeOne CLI（推荐）

```bash
# 1. 克隆仓库
git clone https://github.com/zixilee666-svg/Joan-academic-hub.git
cd Joan-academic-hub

# 2. 安装依赖
npm install

# 3. 安装 EdgeOne CLI
npm install -g @edgeone/cli

# 4. 登录腾讯云账号
edgeone login

# 5. 部署
edgeone pages deploy
```

### 方式二：在 EdgeOne 控制台部署

1. 访问 [EdgeOne Pages 控制台](https://console.cloud.tencent.com/edgeone)
2. 点击「新建站点」
3. 选择「从 GitHub 导入」
4. 授权 GitHub 并选择 `zixilee666-svg/Joan-academic-hub` 仓库
5. 框架预设选择「Vite」
6. 构建命令设置为 `npm run build`
7. 输出目录设置为 `./dist`
8. 点击「部署」

### 部署后配置

部署完成后需要在 EdgeOne 控制台配置以下资源：

#### 1. KV 存储绑定

Edge Functions 使用 KV 存储保存数据，需要：

1. 在 EdgeOne 控制台创建 KV 存储实例
2. 将 KV 存储绑定到 Edge Functions
3. 在环境变量中设置 `KV_STORAGE_ID`

#### 2. 环境变量配置

在 EdgeOne 控制台设置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `JWT_SECRET` | JWT 签名密钥（≥32字符） | `your-secret-key-at-least-32-chars` |
| `KV_STORAGE_ID` | KV 存储实例 ID | `kv-xxxxxxxx` |
| `CLOUD_FUNCTION_TOKEN` | 云函数访问令牌 | `your-token` |
| `AI_API_KEY` | AI 服务 API Key | `sk-xxx` |
| `AI_BASE_URL` | AI API 基础 URL | `https://api.openai.com/v1` |

#### 3. 边缘函数配置

在控制台创建边缘函数：
- 入口文件：`edge-functions/api/[[default]].js`
- 路由：`/api/*`

#### 4. 云函数部署

```bash
cd cloud-functions/api
npm install
# 使用 tcb 或 cf-cli 部署云函数
# 部署到 /api-ai/* 路径
```

## 开发指南

### 环境要求

- **Node.js**: >= 18.0.0（推荐 22.x）
- **npm**: >= 9.0.0
- **Git**: 最新版本

### 本地开发

```bash
# 1. 克隆仓库
git clone https://github.com/zixilee666-svg/Joan-academic-hub.git
cd Joan-academic-hub

# 2. 安装依赖
npm install

# 3. 启动开发服务器（Mock 模式）
npm run dev

# 4. 在浏览器访问 http://localhost:5173
```

### Mock 模式 vs 真实后端模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **Mock 模式** | 使用本地模拟数据，无需后端 | 前端开发、UI 调试 |
| **真实后端模式** | 连接 EdgeOne 后端 API | 端到端测试、生产部署 |

**切换方式**：

编辑 `.env.local` 文件：

```bash
# Mock 模式（默认）
VITE_MOCK_MODE=true

# 真实后端模式
VITE_MOCK_MODE=false
VITE_API_URL=https://your-edgeone-domain.com/api
```

### 构建生产版本

```bash
# 构建
npm run build

# 预览构建结果
npm run preview
```

### 运行 E2E 测试

```bash
# 安装 Playwright 浏览器
npx playwright install

# 运行测试
npm run test:e2e

# 查看测试报告
npm run test:e2e:report
```
